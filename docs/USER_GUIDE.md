# 使用文档

## 1. 安装与启动

### 1.1 获取代码

```bash
git clone https://github.com/ThereWasAYang/trading-assistant.git
cd trading-assistant
```

### 1.2 创建环境

本项目提供 `environment.yml`，环境名称由本地开发者自行指定。下面的 `ta-py312` 只是示例名称，可替换为你的本地环境名。

```bash
conda env create -n ta-py312 -f environment.yml
conda activate ta-py312
```

如果环境已经创建，后续进入项目后直接激活：

```bash
conda activate ta-py312
```

如需手动补装依赖，可执行 `python -m pip install -r requirements.txt pytest`。

### 1.3 启动程序

```bash
python main.py
```

启动后会初始化 SQLite 数据库 `trading_assistant.db`，并写日志到 `logs/`。

## 2. 主界面说明

主窗口由三块组成：

- 左侧分组列表：显示 `持仓中`、`已清仓`、`跟踪中` 和自定义分组。
- 右上股票表格：显示股票代码、名称、行情、买点和提醒状态。
- 右下图表区：显示分时、日线、周线和月线图。

状态栏显示最近刷新时间、持仓盈亏和买点数量。

## 3. 分组管理

### 3.1 预设分组

首次启动会创建：

- `持仓中`: 建议放已经持有的股票。
- `已清仓`: 清仓后的股票。
- `跟踪中`: 观察但未持仓的股票。

### 3.2 新建自定义分组

菜单路径：

```txt
分组 -> 新建自定义分组
```

输入分组名称后，程序调用数据库层新增 `custom` 类型分组。

### 3.3 删除分组

菜单路径：

```txt
分组 -> 删除当前分组
```

只能删除自定义分组，不能删除系统预设分组。删除分组会级联删除该分组中的股票记录，但不会删除交易记录。

## 4. 股票管理

### 4.1 添加股票

菜单路径：

```txt
文件 -> 添加股票
```

可以输入 6 位股票代码或名称关键字。

- 输入 6 位代码时，程序先查本地 `stock_names` 表；没有命中则尝试从 AKShare 同步全市场股票名称。
- 输入名称或关键字时，程序使用 `StockSearchWorker` 异步搜索，优先查本地库，再回退到 AKShare。

添加成功后，程序会启动 `InitialFetchWorker` 拉取日线、周线、月线初始数据并写入数据库。

### 4.2 查看图表

双击股票表格中的一行，会加载该股票图表：

- 分时：调用 AKShare 分时接口。
- 日线/周线/月线：优先从 `MarketDataManager` 读取 SQLite + 内存缓存。

交易时段内，当前图表标签页会按配置定时刷新。

### 4.3 右键菜单

在股票表格行上右键，可执行：

- 查看图表
- 交易记录
- 手动止盈止损设置
- 关闭提醒
- 开启提醒
- 移至已清仓
- 从分组移除

## 5. 交易记录

右键股票行选择 `交易记录`，打开交易记录对话框。

可维护字段：

- 日期
- 类型：买入/卖出
- 价格
- 数量
- 手续费
- 备注

保存后，程序会重新计算持仓摘要：

- 持仓数量
- 持仓成本
- 总买入金额
- 总卖出金额
- 浮动盈亏

如果新增卖出记录后持仓数量小于或等于 0，程序会尝试把该股票从 `持仓中` 自动移动到 `已清仓`。

## 6. 止损止盈提醒

### 6.1 自动逻辑

当前代码中的自动逻辑：

- 止损线：以买入后日线低点向上推进，只上移不下移。
- 止盈线：首次按持仓成本乘以 `TAKE_PROFIT_LIMITUP_RATIO`，默认 1.10；如果检测到短周期顶分型，则锁定为顶分型高点。

交易时段刷新行情后，主窗口会对 `持仓中` 分组股票调用 `AlertEngine` 计算并检查提醒。

### 6.2 手动设置

右键股票行选择 `手动止盈止损设置`。

可以：

- 启用手动止损价。
- 启用手动止盈价。
- 单独清除止损/止盈手动设置。
- 全部恢复自动。

手动设置写入 `settings` 表。程序启动后，`AlertEngine.get_state()` 会从数据库恢复手动设置。

### 6.3 提醒效果

触发止损/止盈后：

- 股票表格行高亮。
- 系统托盘图标闪烁。
- 托盘消息显示提醒内容。

当前表格的 `止损价` 和 `止盈价` 列仍固定显示 `--`，图表止损/止盈线也未从主窗口实际接入。详见 BUG 报告。

## 7. 买点扫描

买点扫描只在交易时段运行，默认每 5 分钟扫描一次。

三项条件满足两项即触发：

- 周线底分型。
- 日线 MACD 金叉。
- 短周期缩量回踩中枢。

触发后：

- 股票表格行显示买点状态并高亮。
- 状态栏显示买点数量。
- 双击该股票会弹出交易纪律对话框。

当前实现中，短周期数据函数名叫 `fetch_30min_kline()`，但实际请求 AKShare 的 60 分钟线。该业务口径需要向原开发者确认。

## 8. 交易纪律

交易纪律默认文本位于：

```txt
resources/discipline.txt
```

买点触发后双击股票会打开交易纪律弹窗。用户需要勾选“已认真阅读并承诺遵守交易纪律”后才能确认关闭。

弹窗支持编辑并保存纪律文本。保存会写入数据库，也会覆盖 `resources/discipline.txt`。如果要区分全局纪律和个股纪律，当前实现需要进一步确认和调整。

## 9. 配置项

主要配置在 [config.py](../config.py)。

常用配置：

- `REALTIME_REFRESH_MS`: 行情刷新间隔，当前 60000 ms。
- `BUYPOINT_SCAN_INTERVAL_MS`: 买点扫描间隔，当前 300000 ms。
- `KLINE_REFRESH_MS`: 当前图表刷新间隔，当前 60000 ms。
- `DB_PATH`: 数据库文件名，当前 `trading_assistant.db`。
- `MA_PERIODS`: 图表均线周期。
- `PRESET_GROUPS`: 预设分组。

## 10. 本地数据文件

运行时生成：

- `trading_assistant.db`: SQLite 数据库。
- `trading_assistant.db-wal` / `trading_assistant.db-shm`: SQLite WAL 文件。
- `logs/`: 日志目录。
- `.pytest_cache/`: pytest 缓存。
- `__pycache__/`: Python 字节码缓存。

这些文件默认被 `.gitignore` 忽略。

## 11. 测试

```bash
python -m pytest -q
```

注意：

- 当前测试会删除项目根目录的 `trading_assistant.db`。
- 当前测试包含真实网络请求，不适合无网络或不稳定网络的 CI 环境。
- 运行 GUI 测试时，PyQt5 可能需要系统图形环境。

## 12. 常见问题

### 搜索不到股票

可能原因：

- 本地 `stock_names` 表为空，且 AKShare 名称同步失败。
- 网络无法访问 AKShare 数据源。
- 输入的代码或名称不在 AKShare 返回范围内。

可尝试重新启动程序，或确认网络后重新添加。

### 图表为空

可能原因：

- 该股票初始 K 线还在后台获取。
- AKShare 接口返回空数据。
- 当前不是交易时段，且数据库中还没有历史缓存。

### 测试通过但功能仍有问题

当前测试覆盖范围有限，尤其没有覆盖做 T 模块资金/仓位守恒、QThread 生命周期、止损/止盈展示接入等实际用户路径。请以 BUG 报告为准继续补测试。
